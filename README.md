# MEUS — 나·너·우리 업무성향 진단

팀빌딩용 자기보고식 업무성향 진단. 4동력(추진·조화·체계·탐구) × 12 facet, 강점을 부족↔적정↔과용 다이얼로 읽는다.

- **나(Me)** — 60문항 진단 → 발표·인쇄용 결과지
- **너(You)** — 두 사람 코드를 맞대 보완·과용조절
- **우리(Us)** — 팀 분포 지도·보완 매트릭스·집단 사각지대

## 쓰는 법

`index.html` 한 파일이 전부다. 더블클릭하면 바로 열리고, GitHub Pages에 올리면 링크로 공유된다. 백엔드 없음 — 결과는 브라우저에 저장되고, 공유는 결과지의 코드/링크로(서버 전송 없음).

라우팅: `#/`(랜딩) · `#/decoder`(해독표) · `#/me`(진단) · `#/you`(2인) · `#/us`(팀) · `#/r/<코드>`(공유 결과).

## 워크숍 흐름

1. 각자 `#/me`에서 ~20분 진단 → 결과지 맨 아래 **코드 복사**
2. 코드를 모아 `#/us`에 붙여넣으면 팀 지도 완성
3. 다 같이 결과지·팀 지도 보며 서로를 읽는다 (40–50분)

## 디자인 토큰

`tokens/`(meus-tokens.css · theme.ts · tailwind.tokens.js)가 색·타입 단일 소스. 값은 `index.html`의 `:root`에 인라인돼 있어 토큰 폴더 없이도 동작한다 — 디자인을 바꾸면 `:root` 블록만 교체.

자기보고식 진단 · 채용·평가가 아닌 상호 이해를 위한 도구.
